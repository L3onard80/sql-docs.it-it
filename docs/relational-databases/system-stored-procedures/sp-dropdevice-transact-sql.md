# <a name="spdropdevice-transact-sql"></a>sp_dropdevice (Transact-SQL)
[!INCLUDE[tsql-appliesto-ss2008-xxxx-xxxx-xxx-md](../../includes/tsql-appliesto-ss2008-xxxx-xxxx-xxx-md.md)]

  Elimina un database o del dispositivo di backup da un'istanza di [!INCLUDE[ssDEversion2005](../../includes/ssdeversion2005-md.md)], l'eliminazione della voce da **sysdevices**.  
   
 ![Icona di collegamento a un argomento](../../database-engine/configure-windows/media/topic-link.gif "Icona di collegamento a un argomento")[Convenzioni della sintassi Transact-SQL](../../t-sql/language-elements/transact-sql-syntax-conventions-transact-sql.md)  
  
## <a name="syntax"></a>Sintassi  
  
```  
  
sp_dropdevice [ @logicalname = ] 'device'   
    [ , [ @delfile = ] 'delfile' ]  
```  
  
## <a name="arguments"></a>Argomenti  
`[ @logicalname = ] 'device'` È il nome logico del database o del dispositivo di backup, come indicato nella **master.dbo.sysdevices.name**. *periferica* viene **sysname**, non prevede alcun valore predefinito.  
  
`[ @delfile = ] 'delfile'` Specifica se il file di dispositivo di backup fisico deve essere eliminato. *delfile* viene **varchar(7)**. Se specificato come **DELFILE**, il file di disco di dispositivo di backup fisico viene eliminato.  
  
## <a name="return-code-values"></a>Valori restituiti  
 0 (esito positivo) o 1 (esito negativo)  
  
## <a name="result-sets"></a>Set di risultati  
 None  
  
## <a name="remarks"></a>Note  
 **sp_dropdevice** non può essere utilizzata in una transazione.  
  
## <a name="permissions"></a>Permissions  
 È richiesta l'appartenenza al ruolo predefinito del server **diskadmin** .  
  
## <a name="examples"></a>Esempi  
 Nell'esempio seguente il dispositivo di dump su nastro `tapedump1` viene eliminato da [!INCLUDE[ssDE](../../includes/ssde-md.md)].  
  
```  
EXEC sp_dropdevice 'tapedump1';  
```
